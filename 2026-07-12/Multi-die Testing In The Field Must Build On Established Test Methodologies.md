# Multi-die Testing In The Field Must Build On Established Test Methodologies

- **日期**: 2026-07-07
- **来源**: Semiconductor Engineering
- **作者**: Laura Peters
- **链接**: https://semiengineering.com/multi-die-testing-in-the-field-must-build-on-established-test-methodologies/
- **分类**: #测试 #先进封装 #多芯片 #可靠性 #Chiplet

## 摘要

器件在"时间零点"正常工作已不再能保证其在整个生命周期内的可靠性。AI革命远超过IC行业为多芯片系统进行充分测试的能力。

## 关键点

1. **系统级测试** — 在系统内进行测试是确保数据中心和汽车硬件长期可靠性的最佳方法
2. **2-die监控策略** — 双芯片监控、测试和修复策略需要修改测试策略和新颖的物理感知凸点修复
3. **微型凸点测试挑战** — 微型凸点（microbump）太小无法直接探针测试，需要使用牺牲焊盘来实现探针接入
4. **测试缺口** — AI的发展速度远超IC行业对多芯片系统在探针测试、最终测试和系统级测试中的全故障机制覆盖能力

## 影响分析

随着Chiplet架构成为主流，测试已从单一的制造环节问题变为贯穿产品全生命周期的挑战。从"测试够不够"到"测试对不对"，行业的测试理念正在发生根本性转变。对于测试设备商和EDA公司来说，系统级测试和在线监控将是新的增长点。
